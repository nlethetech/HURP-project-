# Published dataset exports

Convenience CSV exports of the final analysis panel, for sharing inside this
**private** research repo (e.g. with a supervisor) and for loading in notebooks /
Google Colab. The canonical, fully-reproducible source remains the scripted
pipeline in `src/` — these CSVs are regenerated by
`src/publish/export_dataset_csv.py`.

| File | What | Rows | Size |
|------|------|------|------|
| `hurp_panel_v0.2_full.csv.gz` | The complete panel, all **61 columns**, gzipped | 1,825,173 | ~33 MB |
| `hurp_panel_sample.csv` | Illustrative sample (21 districts × all years, incl. Maiduguri) — renders as a table in GitHub | 777 | ~0.2 MB |

One row per **admin-2 district × year** (1989–2025). Every column is defined in
[`../../docs/CODEBOOK.md`](../../docs/CODEBOOK.md); every source (provider, URL,
license, citation) is in [`../../docs/DATA_SOURCES.md`](../../docs/DATA_SOURCES.md).

## ⚠️ License — keep this private

The full export includes the **ACLED** columns (`acled_*`). ACLED's EULA permits
the licensee to use derived data for their own research but **forbids public
redistribution**. Therefore:

- **This repository must stay PRIVATE.** Do not make it public while these CSVs
  (or any `acled_*` data) are committed.
- Collaborators who need the data should ideally re-pull ACLED with their **own**
  myACLED credentials (`src/acquisition/10_download_acled.py`), per the EULA.
- All other columns are openly licensed (CC BY 4.0: UCDP GED, World Bank, SPAM,
  GDHY, CHIRPS, geoBoundaries; Powell & Thyne for coups). If you ever need a
  **publicly shareable** export, regenerate it without the `acled_*` columns.

Required attributions (see `docs/DATA_SOURCES.md` for full citations): UCDP GED
26.1; ACLED (Raleigh et al. 2023; accessed 2026-06-15); Powell & Thyne 2011;
World Bank WDI & OGHIST; IFPRI SPAM 2020; GDHY (Iizumi 2019); CHIRPS; geoBoundaries.

## Loading

```python
import pandas as pd
# Full panel (pandas reads .gz transparently):
df = pd.read_csv("hurp_panel_v0.2_full.csv.gz")
# or the small sample:
df = pd.read_csv("hurp_panel_sample.csv")
```

`NaN` is meaningful — it marks *not observed*, never zero. In particular `acled_*`
is `NaN` outside ACLED's staggered country coverage, and the World Bank `wb_*`
columns are `NaN` where the Bank has no observation. See the codebook before
treating any gap as a zero.

A ready-to-run analysis notebook is in
[`../../notebooks/hurp_analysis_colab.ipynb`](../../notebooks/hurp_analysis_colab.ipynb).
